
Task 2. Cтворити базовий клас CurrencyAPI, який:
    Зберігає API-ключ та ім'я API.
    Має загальний словник cache для кешування відповідей.

    Має методи:
        set_to_cache(key, value) — зберігає дані в кеш із повідомленням у консоль.
        get_from_cache(key) — отримує дані з кешу за ключем.
        status_code(status_code) — обробляє статуси HTTP-відповідей:
            200 → все добре
            404 → виводити повідомлення про відсутність ресурсу
            428 → зробити паузу на 4 секунди
        __repr__() — повертає короткий опис об'єкта із зазначенням імені API та дати.
Task 3. Створити клас MonoAPI, який наслідується від CurrencyAPI.
    Додати:
        Поле url для зберігання адреси API.
        Поле data для збереження отриманих даних.

    Реалізувати методи:
        get_currency(currency):
        Перевірити, чи є курс у кеші.
        Якщо є — повернути кешовану відповідь.
        Якщо нема — зробити запит до API через requests.get.
        Перевірити статус коду через self.status_code().
        Якщо все добре — зберегти відповідь у self.data.
        Викликати метод parsing_data(currency) для обробки відповіді.

    parsing_data(currency):
        Перебрати список даних (self.data).
        Знайти об'єкт із потрібною валютою (порівнювати currencyCodeA).
        Витягти курс купівлі (rateBuy) або продажу (rateSell).
        Зберегти знайдений курс у кеш через set_to_cache.
        Повернути результат.
